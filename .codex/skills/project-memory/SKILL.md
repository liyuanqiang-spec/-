# project-memory

## Purpose

Keep the project coherent across many GPT and worker turns.

## Maintain

- Project goal.
- Current working phase.
- Latest completed task.
- Open blocker.
- Next safe action.
- Files that matter.

## Rules

- Update memory after meaningful project changes.
- Do not store private credentials or local-only private paths.
- Keep memory concise and actionable.
